Context

Problema de Negócio

Instituições financeiras precisam reduzir perdas relacionadas à inadimplência sem comprometer excessivamente a aprovação de crédito. O objetivo é identificar clientes com maior probabilidade de default para apoiar decisões de concessão e limites.

Engineering

Pipeline de Machine Learning

1. Análise Exploratória & Pré-processamento

Tratamento de outliers, encoding de variáveis categóricas e análise de correlação entre score de mercado e inadimplência.

2. Modelagem & Ajuste

Treinamento de XGBoost, Random Forest e Regressão Logística. Otimização do threshold para 0.3 visando maximizar o Recall.

Strategy

Decisões Estratégicas

Formulação do Problema

O problema foi convertido de regressão para classificação binária (threshold 7), alinhando o modelo à decisão real de concessão de crédito.

Foco no Recall

Priorizamos o Recall para minimizar Falsos Negativos, já que o custo de inadimplência é significativamente maior que o custo de oportunidade de uma venda perdida.

Validação Robusta

Utilização de Stratified K-Fold para garantir estabilidade estatística e evitar overfitting em classes desbalanceadas.

📊 Dataset

Base de dados de uma fintech de crédito pessoal com variáveis como score de mercado, alavancagem, saldo rotativo e consultas ao bureau.

💡 Strategic Decisions

  • Formulation as a binary classification alined with actual lending policies
  • Recall focus reduced potential default exposure by 15%
  • Score de mercado revealed high collinearity with target, requiring PCA
  • Model interpretability used for regulatory compliance documentation
